The Gap in Go Tooling

Every mainstream language has a code transformation ecosystem. Java has OpenRewrite and Error Prone. C# has Roslyn analyzers with code fixes. Python has Ruff with auto-fix. TypeScript has codemods and ESLint with --fix.

Go has staticcheck and golangci-lint. Both are excellent at finding problems. Neither fixes them. Every diagnostic is a ticket for a human. Across hundreds of repositories, that doesn't scale.

We built the missing piece: a recipe library that reads Go source code at the AST level, applies deterministic transformations, and produces source-faithful output. Every comment preserved. Every formatting choice respected. The diff is exactly what the recipe changed and nothing else.

Recipe Categories

Simplification (61 recipes)

The largest category. These recipes modernize Go code by replacing deprecated or verbose patterns with cleaner standard library alternatives.

Deprecated API migration (7 recipes). The ioutil package was deprecated in Go 1.16. Every call to ioutil.ReadAll, ioutil.ReadFile, ioutil.WriteFile, ioutil.TempDir, ioutil.TempFile, ioutil.NopCloser, and ioutil.ReadDir has a direct replacement in io or os. Our recipes handle all seven, including the import change.

Index-to-Contains family (12 recipes). The pattern strings.Index(s, sub) != -1 should be strings.Contains(s, sub). We handle this for strings.Index, strings.IndexAny, strings.IndexRune, bytes.Index, bytes.IndexAny, bytes.IndexRune — both positive and negative comparisons, including the >= 0 and < 0 variants.

Boolean simplification (8 patterns). x == true → x. x == false → !x. All four operators, both operand orders. Implemented as two template recipes with four before-patterns each — our Refaster-style "anyOf" mechanism.

Go 1.21+ adoption. sort.Sort(sort.IntSlice(s)) → sort.Ints(s). math.Min(a, b) → min(a, b). log.Println(x) → slog.Info(x) for structured logging migration.

Error Handling (27 recipes)

Go error handling is the single largest source of code quality issues in Go codebases. We've built the most comprehensive automated error handling remediation available for Go.

errors.Is migration. Direct error comparison (err == io.EOF) fails with wrapped errors. We transform these to errors.Is(err, io.EOF) for eight common sentinels: io.EOF, sql.ErrNoRows, http.ErrServerClosed, net.ErrClosed, os.ErrInvalid, context.Canceled, context.DeadlineExceeded, and package-qualified sentinels via FieldAccess matching.

errors.As migration. The common pattern if myErr, ok := err.(*MyError); ok { ... } should use errors.As for wrapped error support. Our recipe transforms this to var myErr *MyError; if errors.As(err, &myErr) { ... } using Block-level visiting to split one statement into two.

Sentinel extraction. Inline errors.New("not found") calls inside functions can't be compared with errors.Is. Our recipe extracts them to package-level variable declarations: var ErrNotFound = errors.New("not found"). This uses CompilationUnit-level visiting to modify the file's top-level declarations while replacing inline calls with references.

Error wrapping. Three recipes address error context:

  • return err → return fmt.Errorf("funcName: %w", err) — adds function name as context
  • return nil, err → return nil, fmt.Errorf("funcName: %w", err) — multi-return variant
  • fmt.Errorf("...: %s", err) → fmt.Errorf("...: %w", err) — fixes the format verb

Swallowed error recovery. if err != nil { return } (bare return that silently swallows the error) → if err != nil { return err }. Empty error check blocks (if err != nil { }) get filled with return err.

Style & Security (72 recipes)

Resource management. Nine "ensure" recipes detect resource creation (os.Open, db.Query, time.NewTicker, etc.) and auto-insert the corresponding defer Close/Stop/Rollback statement. This uses Block-level visiting to scan for the creation assignment and splice the defer statement after it, with idempotency checks to avoid double-deferring.

Security remediation. InsecureSkipVerify: true → false. File permissions 0777 → 0755. Hardcoded credentials → os.Getenv("VAR_NAME") with SCREAMING_SNAKE_CASE name generation. import . "pkg" → import "pkg". Regex patterns in interpreted strings → raw strings.

Guard clause refactoring. if err == nil { <deeply nested> } → if err != nil { return }\n<less nested>. Inverts conditions to enable early returns, reducing nesting depth. Includes automatic dedentation of the lifted code.

Performance (20 recipes)

Regex hoisting. regexp.MustCompile("pattern") inside a loop is compiled every iteration. Our recipe hoists the compile call before the loop and replaces the in-loop usage with a variable reference. For regexp.Compile (two-return), it promotes to MustCompile since the pattern is a compile-time literal.

strings.Builder insertion. s += expr inside a loop becomes three statements: var builder strings.Builder before the loop, builder.WriteString(expr) inside, s = builder.String() after. Including automatic import "strings" addition.

Defer-in-loop IIFE wrapping. defer f.Close() inside a loop accumulates defers until function exit. Our recipe wraps the loop body in an immediately-invoked function literal so defers run per-iteration.

Map preallocation. make(map[K]V) inside a loop creates a new map each iteration. Our recipe hoists the allocation before the loop and inserts clear(m) at the start of each iteration.

Redundancy (23 recipes)

Dead code removal: unreachable statements after return, empty for {} loops, empty switch {}, empty default: cases, self-assignments (x = x), and *&x double-deref.

Control flow simplification: if true { body } → body. if false { } → deleted. if len(s) > 0 { for range s } → for range s. if cond { return } else { body } → if cond { return }\nbody. go func() { f() }() → go f(). for k := range m { delete(m, k) } → clear(m).

Naming (9 recipes)

All naming recipes are now transformational. func (self *Foo) Bar() → func (f *Foo) Bar(). func (u *User) GetName() → func (u *User) Name(). MAX_BUFFER_SIZE → MaxBufferSize. var notFound = errors.New(...) → var ErrNotFound = errors.New(...). func f(context context.Context) → func f(ctx context.Context) with body-wide rename.

How It Works

Template Recipes

For straightforward before→after patterns, we built a Refaster-like declarative API. A recipe author writes the before pattern and after pattern as Go code strings with named captures. The framework parses both through scaffold wrappers, matches the before against every AST node, and applies the after with captured values substituted.

s := template.Expr("s")
r := template.NewRecipe(
    template.WithBefore(fmt.Sprintf(`fmt.Sprintf("%%s", %s)`, s), template.Imports("fmt")),
    template.WithAfter(fmt.Sprintf(`%s`, s)),
    template.WithCaptures(s),
)

Multiple before-patterns (Refaster's "anyOf") are supported — first match wins. This is how the boolean simplification recipe handles eight different input patterns with a single after pattern.

Block-Level Visiting

The breakthrough that unlocked our highest-value transforms. Instead of visiting individual nodes (MethodInvocation, If, etc.), we visit Block — the brace-delimited list of statements. This enables:

  • Statement insertion — add defer f.Close() after a file open
  • Statement hoisting — move regexp.MustCompile before a loop
  • Statement splitting — split if myErr, ok := err.(*T); ok into var + if
  • Redundant else removal — extract else body as sibling statements
  • IIFE wrapping — wrap loop body in func() { ... }()

CompilationUnit-Level Visiting

For file-wide transforms like error sentinel extraction, we visit the CompilationUnit (the whole file). A two-pass approach: first collect inline errors.New calls from function bodies, then add var declarations at the top and replace the inline calls with references.

MethodMatcher

AspectJ-style pattern matching for method invocations. "fmt Sprintf(..)" matches any fmt.Sprintf call regardless of arguments. "time.Time Sub(time.Time)" matches Sub with one time.Time argument. Supports glob wildcards: * for single name, *..* for any type in any package.

Markup Severity

Recipes that detect but can't auto-fix use severity-appropriate diagnostics: MarkupWarn for definite issues (SQL injection, race conditions, panic), MarkupInfo for suggestions (consider X, ensure Y). This replaces the generic SearchResult marker with actionable severity levels.

Where We Go From Here

Go Version Migration Recipes

Composite recipes for Go 1.18→1.21→1.22→1.23, following the C# pattern of UpgradeToDotNet10. Each version adds new stdlib APIs, deprecates old ones, and introduces language features (generics, range-over-func). Recipes automate the adoption path.

Framework Migrations

gin→chi, logrus→slog, testify assertion patterns, gRPC version upgrades. The template recipe mechanism makes these straightforward to author — most framework migrations are method-to-method API mappings.

Module Dependency Management

Update go.mod, resolve breaking changes across the dependency graph, handle version conflicts. This connects to the existing Moderne platform dependency management capabilities.

Type-Attributed Transforms

Deeper type information enables removing truly redundant type conversions, smarter error handling (knowing a function returns error), and interface-aware refactoring. The Go parser already performs type attribution via go/types — the recipes need to consume it more aggressively.

Multi-File Refactoring

Rename methods across all callers. Extract interfaces. Move packages. These require the Moderne platform's cross-repository orchestration — the recipe sees one file at a time, but the platform coordinates across all files in all repositories.

Harness Comparison

Automated comparison against staticcheck and golangci-lint on real repositories, following the C# pattern where every recipe is validated against its Roslyn analyzer equivalent. The infrastructure exists (diagnostic mapping, working set of repos). The automation pipeline is next.
