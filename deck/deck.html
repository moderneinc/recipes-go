The Problem

Go doesn't have automated remediation.

Java has OpenRewrite. C# has Roslyn code fixes. Python has Ruff with auto-fix. Go has… linters that report problems and leave the fixing to you.

staticcheck and golangci-lint find issues. They don't fix them. Every finding is a manual task — triage, understand, edit, test, repeat. Across hundreds of repositories, that manual work doesn't scale.

  • staticcheck — flags deprecated APIs. Doesn't migrate them.
  • golangci-lint — finds naming violations. Doesn't rename.
  • go vet — catches bugs. Doesn't fix them.
  • Manual remediation — every finding requires a human. Every repo requires a PR.

The SDK

Full Go language support in OpenRewrite.

  • GoTemplate + GoPattern — Refaster-like declarative recipe API. Define before/after patterns, the framework handles matching and replacement.
  • MethodMatcher — AspectJ-style method matching: "fmt Sprintf(..)" matches any fmt.Sprintf call.
  • TypeUtils — Type hierarchy checking, assignability, error/string detection.
  • Block-level visiting — Insert defer statements, hoist code before loops, splice multiple statements.
  • CompilationUnit-level visiting — Add package-level declarations, modify imports.
10-Line Recipe
s := template.Expr("s")
r := template.NewRecipe(
  template.WithBefore(
    fmt.Sprintf(
      `fmt.Sprintf("%%s", %s)`, s),
    template.Imports("fmt")),
  template.WithAfter(
    fmt.Sprintf(`%s`, s)),
  template.WithCaptures(s),
)

What used to take 50+ lines of manual visitor code now takes 10 lines of declarative pattern matching.
